简单理解OAuth 2.0授权机制

2023-01-09 14:39
快递员问题

例如我住在一个大型的居民小区，小区有门禁系统，进入小区需要输入密码，我经常网购喝外卖，快递员进入小区需要通过门禁，如果我把门禁密码告诉每个快递员，他们就拥有了和我相同的权限不太合适，如果我想取消快递员进入小区的权利，我就得更改密码这样还得通知其他快递员，这样就引出了OAuth2.0的授权机制，门禁上添加一个获取授权，会给一个有效期的令牌。
互联网上最常见的OAuth2.0就是各种第三方登录

四种模式

http://itboyhub.com/2021/01/26/spring-security-oauth2-login/
1.授权码模式（authorization_code）: 直接从后台发送到授权服务器上很常用，减少access_token泄漏风险
2.简化模式: （implicit）前台发送不安全
3.密码模式 （password）
4.客户端模式（client_credentials）

JWT

-全称是 Json Web Token ， 是一种 JSON 风格的轻量级的授权和身份认证规范，可实现无状态、分布式的 Web 应用授权
-可以实现无状态登录
